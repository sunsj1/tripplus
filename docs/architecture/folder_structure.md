## Rules for adding things

| Adding | Goes in |
|---|---|
| A new user-facing concept | New folder under `lib/features/<name>/` |
| A new Freezed entity used by exactly one feature | `lib/features/<feature>/domain/models/` |
| A new Freezed entity used by 2+ features | `lib/core/domain/` |
| A new SDK client (Google API, OCM, etc.) | `lib/core/services/` |
| A reusable widget used by 2+ features | `lib/core/widgets/` |
| A widget used only by one feature | `lib/features/<feature>/presentation/widget/` |
| An offline queue | `lib/features/<feature>/data/local_db/` (or `data/<feature>_queue.dart`) |
| A theme/color constant | `lib/core/theme/` |
| A pure helper | `lib/core/utils/` |
| Generated files (`*.freezed.dart`, `*.g.dart`) | Next to their source, committed to git |

---

## Naming conventions

- File names: `snake_case.dart`
- Provider files: end with `_providers.dart`
- Controller class names: end with `Controller`
- UI state class names: end with `UiState` (Freezed sealed union)
- Repository class names: end with `Repository`
- DTO class names: end with `Dto`
- Key helpers: `<feature>_key.dart` (one function exported)
